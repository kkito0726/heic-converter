# PRD: heic-converter

.heicファイルを他形式の画像に変換するリッチな対話型CLIツール

## 1. 背景・課題

- iPhone / iPad で撮影した写真はデフォルトで HEIC (High Efficiency Image Container) 形式で保存される
- HEIC は Apple 以外のデバイス・古いソフトウェア・多くの Web サービスでそのまま閲覧/アップロードできないことが多い
- 既存の変換手段(オンラインコンバータ、ImageMagick 等)は「アップロードが必要」「外部ツールのインストールが必要」「操作が煩雑」といった課題がある

## 2. ゴール

| ゴール | 内容 |
|---|---|
| 手軽さ | バイナリを1つ置くだけで動作。外部ツール・ライブラリのインストール不要 |
| 対話性 | 引数なしで起動しても、対話形式で迷わず変換を完了できる |
| 一括処理 | フォルダを渡せば配下の HEIC をまとめて変換できる |
| 体験 | 配色・アスキーアート・進捗表示など、使っていて楽しいリッチな CLI |

### 非ゴール(スコープ外)

- HEIC **への**変換(逆方向)
- 画像編集機能(リサイズ・トリミング・画質調整)※将来の拡張候補
- GUI / Web UI(ただし将来 API として起動できるよう設計上は考慮する)
- 動画コンテナ(HEVC 動画など)の変換

## 3. ターゲットユーザー

- iPhone の写真を PC (Windows / Linux / Mac) で扱いたい一般ユーザー〜開発者
- オンラインコンバータに写真をアップロードしたくない(プライバシー意識のある)ユーザー
- 大量の HEIC をまとめて変換したいユーザー

## 4. 機能要件

### FR-1: 入力パスの解釈

- **ファイルパス**が渡された場合: その HEIC ファイル1件を変換する
- **フォルダパス**が渡された場合: フォルダ内の `.heic` / `.heif` ファイルを**すべて**変換する
  - サブフォルダを再帰的に処理するかは対話 or フラグで選択できる
- パスが存在しない / HEIC が1件も見つからない場合は分かりやすいエラーメッセージを表示する
- 拡張子の大文字小文字(`.HEIC` 等)は区別しない

### FR-2: 対話モード(デフォルト)

引数なし、またはパスのみで起動した場合は対話形式で進行する。

1. **起動画面**: アスキーアートのロゴ + ツール概要を表示
2. **パス入力**: 引数でパスが渡されていなければ、入力プロンプトで受け付ける
3. **出力形式の選択**: チェックボックス(複数選択可)で変換先の拡張子を選ぶ
   - 対応形式: `jpg` / `png` / `webp` / `tiff` / `bmp` / `gif`
   - 複数選択した場合、1つの HEIC から選択した全形式のファイルを生成する
4. **オプション確認**: 出力先ディレクトリ、上書き可否などを確認
5. **変換実行**: プログレスバー + ファイルごとの成否をリアルタイム表示
6. **結果サマリ**: 成功/失敗件数、出力先、失敗ファイルの一覧と理由を表示

### FR-3: 非対話モード(フラグ指定)

CI やスクリプトからも使えるよう、フラグが揃っていれば対話をスキップして即実行する。

```
heic-converter <path> --format jpg,png --output ./converted --recursive --overwrite
```

- `--format, -f` : 変換先形式(カンマ区切りで複数指定可)
- `--output, -o` : 出力先ディレクトリ(デフォルト: 入力ファイルと同じ場所)
- `--recursive, -r` : フォルダを再帰的に処理
- `--overwrite` : 既存ファイルを上書き(デフォルトは確認 or スキップ)
- `--quality, -q` : JPEG/WebP の品質 (1-100、デフォルト 90)

### FR-4: 変換仕様

- 出力ファイル名: 元ファイル名の拡張子を変換先に差し替える(`IMG_0001.heic` → `IMG_0001.jpg`)
- EXIF の Orientation を反映し、画像の向きが正しく出力される
- 複数ファイルは並列で変換し、マルチコアを活用する

### FR-5: リッチな見た目

- 起動時にアスキーアートのロゴを表示
- 統一されたカラーテーマ(選択中の項目のハイライト、成功=緑 / 失敗=赤 など)
- 変換中はプログレスバーとスピナーで進捗を可視化
- ターミナル幅に応じたレイアウト調整

## 5. 非機能要件

| 項目 | 要件 |
|---|---|
| 配布形態 | **単一バイナリ**。cgo 不使用(`CGO_ENABLED=0`)でビルドでき、実行環境に外部依存がない |
| 対応 OS | macOS / Linux / Windows (amd64 / arm64) |
| 性能 | 並列変換により、100枚程度の一括変換が実用的な時間で完了する |
| 信頼性 | 1ファイルの変換失敗が全体を止めない(失敗はスキップして最後にレポート) |
| 安全性 | デフォルトでは既存ファイルを黙って上書きしない |
| 拡張性 | デコーダ/エンコーダ/入力層を差し替え可能な設計(クリーンアーキテクチャ) |

## 6. 将来構想

- 対応形式の追加(avif など)
- リサイズ・品質最適化などの変換オプション拡充
